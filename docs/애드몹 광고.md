# 광고 적용 가이드

Google AdMob을 Android와 iOS 앱에 연동하는 과정은 크게 \*\*1. AdMob 콘솔 설정(웹사이트)\*\*과 **2. 앱 프로젝트 설정(코드 및 환경)** 두 단계로 나뉩니다.

실수하기 쉬운 '앱 ID'와 '광고 단위 ID'의 차이를 명확히 구분하며 단계별로 설명해 드리겠습니다.

-----

### 1단계: AdMob 콘솔 설정 (웹사이트)

코드를 작성하기 전에 구글 AdMob 웹사이트에서 앱을 등록하고 ID를 발급받아야 합니다.

1.  **AdMob 가입 및 로그인:** Google AdMob에 접속하여 계정을 생성합니다.

2.  **앱 추가 (App Creation):**

      * 대시보드에서 '앱 추가'를 클릭합니다.
      * **플랫폼 선택:** Android와 iOS를 각각 별도로 등록해야 합니다 (즉, 앱이 하나라도 AdMob 상에서는 Android용, iOS용 2개를 만듭니다).
      * **앱 스토어 등록 여부:** 아직 출시 전이라면 "아니요"를 선택하고 나중에 스토어에 출시 후 연결할 수 있습니다.
      * **결과:** \*\*앱 ID (App ID)\*\*가 생성됩니다. (`ca-app-pub-xxxxxxxxxxxxxxxx~xxxxxxxxxx` 형식)
      * *주의: 이 ID는 앱의 설정 파일에 들어가는 중요한 신분증입니다.*

3.  **광고 단위 만들기 (Create Ad Unit):**

      * 앱 개요 화면에서 '광고 단위 추가'를 누릅니다.
      * **형식 선택:** 배너(Banner), 전면(Interstitial), 리워드(Rewarded) 중 원하는 형식을 선택합니다.
      * **결과:** \*\*광고 단위 ID (Ad Unit ID)\*\*가 생성됩니다. (`ca-app-pub-xxxxxxxxxxxxxxxx/xxxxxxxxxx` 형식)
      * *주의: 이 ID는 실제 화면에 광고를 띄울 때 코드에 입력하는 주소입니다.*

-----

### 2단계: 프로젝트 환경 설정 (Native 설정)

이제 각 플랫폼의 프로젝트 설정 파일에 **앱 ID**를 등록해야 합니다. (Flutter나 React Native를 쓰더라도 이 네이티브 설정은 필수입니다.)

#### **Android 설정**

1.  **Gradle 설정 (`build.gradle`):**
      * `dependencies` 항목에 `play-services-ads` 라이브러리를 추가합니다.
2.  **Manifest 수정 (`AndroidManifest.xml`):**
      * `<application>` 태그 내부에 `<meta-data>`를 추가하여 **앱 ID**를 입력합니다.
    <!-- end list -->
    ````xml
    <meta-data
        android:name="com.google.android.gms.ads.APPLICATION_ID"
        android:value="ca-app-pub-xxxxxxxxxxxxxxxx~xxxxxxxxxx"/> ```

    ````

#### **iOS 설정**

1.  **Info.plist 수정:**
      * `Info.plist` 파일을 열어 `GADApplicationIdentifier` 키를 추가하고 값으로 **앱 ID**를 입력합니다.
      * (선택 사항) 사용자 추적 권한(ATT)이 필요한 경우 `NSUserTrackingUsageDescription` 문구도 추가해야 합니다.
    <!-- end list -->
    ```xml
    <key>GADApplicationIdentifier</key>
    <string>ca-app-pub-xxxxxxxxxxxxxxxx~xxxxxxxxxx</string>
    ```

-----

### 3단계: SDK 초기화 및 광고 로드 (코드 구현)

앱이 실행될 때 AdMob SDK를 깨우고, 광고를 불러오는 과정입니다.

1.  **SDK 초기화 (Initialization):**

      * 앱이 시작되는 시점(보통 `main` 함수나 `Launch` 시점)에 SDK를 초기화하는 코드를 한 번 실행해야 합니다.
      * `MobileAds.initialize()` (Android/Flutter) 또는 `GADMobileAds.sharedInstance().start()` (iOS) 등의 함수를 사용합니다.

2.  **광고 로드 (Load Ad):**

      * 광고를 보여주고 싶은 화면에서 **광고 단위 ID**를 사용하여 광고를 로드합니다.
      * 이때, **테스트용 ID**를 먼저 사용하는 것이 필수입니다. (개발 중에 실제 ID를 쓰면 계정이 정지될 수 있습니다.)

3.  **광고 표시 (Show Ad):**

      * 로드 완료(Callback) 신호를 받으면 화면에 광고를 띄웁니다.

-----

### 4단계: 필수 확인 사항 (중요)

> **⚠️ 주의: 계정 정지를 막기 위해 반드시 지켜야 할 규칙입니다.**

1.  **테스트 광고 사용:** 개발 및 테스트 중에는 구글이 제공하는 \*\*[테스트용 광고 단위 ID]\*\*를 사용해야 합니다. 본인의 실제 광고를 직접 클릭하면 부정 행위로 간주되어 계정이 영구 정지될 수 있습니다.
2.  **app-ads.txt 설정:** 앱을 스토어에 출시한 후, AdMob 콘솔에서 제공하는 `app-ads.txt` 내용을 본인의 개발자 웹사이트 도메인 루트에 업로드해야 수익 창출이 승인됩니다.
3.  **개인정보 동의 (UMP):** 유럽(GDPR) 등 특정 국가를 타겟팅한다면 구글의 UMP(User Messaging Platform) SDK를 통해 사용자 동의를 받는 팝업을 띄워야 합니다.
